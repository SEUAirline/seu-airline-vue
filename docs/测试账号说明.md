## 注意事项

1. **Mock数据说明**
   - 当前使用的是Mock数据，所有数据存储在内存中
   - 刷新页面后，除了已登录的用户信息外，其他数据会重置
   - 环境变量 `VITE_USE_MOCK=true` 时使用Mock数据

2. **切换到真实后端**
   - 修改 `.env.development` 文件
   - 设置 `VITE_USE_MOCK=false`
   - 确保后端服务已启动（默认端口8080）
   - 重新启动前端开发服务器

3. **常见问题**
   - 如果登录失败，请检查：
     - 用户名和密码是否正确
     - 浏览器控制台是否有错误信息
     - Mock服务是否正常启动（查看终端日志）
   
4. **开发建议**
   - 开发阶段建议使用Mock数据，避免依赖后端
   - 测试完整流程时再切换到真实后端
   - 可以在 `mock-data.ts` 中添加更多测试数据
