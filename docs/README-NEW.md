## 📋 文档使用指南

### 新成员快速上手

**第1步:** 阅读核心文档
```
开发计划.md → 了解当前开发任务
开发记录.md → 了解项目进展
调试说明.md → 学习调试方法
```

**第2步:** 理解系统架构
```
备注文档.md → 查看系统架构设计
module-design.md → 了解技术架构和模块设计
```

**第3步:** 配置测试环境
```
调试说明.md → 配置环境并测试功能
TESTING-GUIDE.md → 详细测试指南
```

**第4步:** 开始开发
```
开发计划.md → 查看待开发任务
开发记录.md → 参考已完成功能
```

---

### 开发人员日常使用

#### 每日开发流程

1. **查看当日计划**
   - 打开 `开发计划.md`
   - 确认今日任务和优先级

2. **开发过程中**
   - 参考 `备注文档.md` 了解接口规范
   - 参考 `module-design.md` 了解技术选型

3. **测试功能**
   - 使用 `调试说明.md` 进行功能测试
   - 记录发现的问题

4. **提交代码前**
   - 更新 `开发记录.md`
   - 记录开发总结

---

### 项目经理使用

#### 进度跟踪

1. **查看最新进度**
   - `开发记录.md` - 详细进度报告
   - `开发计划.md` - 当前开发计划

2. **评估完成度**
   - `开发记录.md` - 查看完成度统计
   - `开发计划.md` - 对照计划评估进度

3. **规划下一步**
   - `开发计划.md` - 制定新的开发计划

## 💡 文档整理说明

本次文档整理将原有的41个文档整合为7个核心文档，主要改进：

1. **信息集中化** - 相关内容整合到一起，避免分散
2. **结构清晰化** - 4个核心文档 + 3个辅助文档
3. **查找便捷化** - 不再需要在多个文档间跳转
4. **维护简单化** - 减少文档数量，降低维护成本

**整合原则:**
- 开发计划类 → `开发计划.md`
- 进度报告类 → `开发记录.md`
- 调试修复类 → `调试说明.md`
- 设计参考类 → `备注文档.md`

**保留原则:**
- 独立性强的文档保留（如TESTING-GUIDE.md）
- 技术参考文档保留（如module-design.md）
- 需求文档保留（如requirement.md）
